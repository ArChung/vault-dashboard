# /kid 規則

紀錄阿忠家兩個小孩（子誼、子寬）的日常與金句。

## 兩個小孩

- **子誼**：姊姊
- **子寬**：弟弟

檔案分別存在：
- `$VAULT/Kids/子誼.md`
- `$VAULT/Kids/子寬.md`

## 判斷對象

1. 使用者明確說「子誼...」或「子寬...」→ 對應那一個
2. 說「姊姊」→ 子誼；說「弟弟」→ 子寬
3. 同一句話提到兩個小孩 → **兩個檔案都更新**（各寫對應的部分）
4. 只說「小孩」「孩子」沒指名 → 反問一句「是子誼還是子寬？」

## 檔案標準結構（首次建檔）

```markdown
# 子誼（或 子寬）

## 金句集
<最新的金句在最上方>

## 日常記錄
<按時間倒序，最新的在最上方>
```

**重要**：金句集與日常記錄都是**倒序**，最新的放最上面。這樣打開檔案就看到最新的。

## 每筆記錄格式

### 日常記錄區

```markdown
### YYYY-MM-DD HH:MM
<整理過的記錄內容>
```

### 金句集區

```markdown
### YYYY-MM-DD HH:MM
> 「<原話>」
<可選：簡短情境註解>
```

## 金句判斷規則

一段話裡**同時**有日常跟金句時，兩區都要寫：

- **金句定義**：
  - 小孩講的有引號內容（「」『』""）
  - 使用者明確說「這句話很有趣」「金句」「記一下這句」
  - 童言童語式的經典句子
- **抽取行為**：
  - 金句區只放引號那一句（加簡短情境）
  - 日常記錄區保留完整敘述（包含金句本身）
  - 同一個時間戳，兩區用同一個 `YYYY-MM-DD HH:MM`

## 寫入流程

1. `date` 取得 `YYYY-MM-DD HH:MM`
2. 判斷對象（子誼/子寬/兩個）
3. 對每個對象：
   - 檢查 `$VAULT/Kids/XXX.md` 是否存在
   - **不存在** → 用標準結構建檔
   - **存在** → Read 出來，Edit 在 `## 金句集` 跟 `## 日常記錄` 下面**最上方**插入新記錄
4. 抽金句：判斷有沒有符合金句條件的內容，有就同時寫到金句集
5. commit + push

## 插入位置注意

Edit 時要找正確的插入點：

```
## 金句集
<在這裡插入新金句，讓它變成最上面>

### 舊的金句...
```

```
## 日常記錄
<在這裡插入新記錄，讓它變成最上面>

### 舊的記錄...
```

實作方式：用 `## 金句集\n` 當 old_string 前綴，接 `\n### <新時間戳>\n<新內容>\n` 再接原本下面的內容。

## 範例

輸入：「/kid 子誼今天說『我長大要當太空人』，吃飯吃超慢又挑食」

寫入（首次建檔 `Kids/子誼.md`）：

```markdown
# 子誼

## 金句集

### 2026-04-09 18:30
> 「我長大要當太空人」

## 日常記錄

### 2026-04-09 18:30
今天說「我長大要當太空人」。吃飯超慢又挑食。
```

下次追加（輸入「/kid 子誼今天學校表演很棒」）：

```markdown
# 子誼

## 金句集

### 2026-04-09 18:30
> 「我長大要當太空人」

## 日常記錄

### 2026-04-10 20:00
今天學校表演很棒。

### 2026-04-09 18:30
今天說「我長大要當太空人」。吃飯超慢又挑食。
```

（新的在上，金句集這次沒有更新因為沒有引號內容）

## 雙更新範例

輸入：「/kid 子誼跟子寬搶玩具，子寬一直說『這是我的』」

- 更新 `Kids/子寬.md`：
  - 金句集加：「這是我的」
  - 日常記錄加：跟姊姊搶玩具，一直說「這是我的」
- 更新 `Kids/子誼.md`：
  - 金句集不動（沒有子誼講的話）
  - 日常記錄加：跟弟弟搶玩具

commit 訊息可以合併：`子誼子寬：搶玩具事件，子寬新金句「這是我的」`

## Commit 訊息

格式：`<對象>：<事件短摘要>`

範例：
- `子誼：新增金句「我要當太空人」`
- `子寬：學會自己穿襪子`
- `子誼子寬：兩人搶玩具`
