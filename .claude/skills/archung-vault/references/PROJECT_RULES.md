## 專案檔 metadata 格式

```markdown
# 專案名稱
> 狀態：進行中 / 等待 / 已完成
> 客戶：XXX
> 窗口：XXX(聯絡方式)
> 報價：XXX 未稅 / 未報價 / 未成案
> 已開發票：XXX          ← 有才寫
> 分潤：XXX              ← 有才寫
> 開始日期：YYYY-MM-DD

## 最新結論
（一段話總結目前狀態和最新決定，每次有重大進展就覆寫）

## 進度紀錄
- YYYY-MM-DD 發生了什麼

## 待辦
- [ ] 還沒做的事
- [x] 做完的事

## 備註
（合約細節、報價單明細、執行團隊等不常變動的資訊）
```

**重點**：
- 沒有的欄位**省略整行**，不要寫「—」或留空
- 尾款**不寫**，index.html 會自動從 報價 - 已開發票 算出
- 「最新結論」是**覆寫式**，每次重大進展整段重寫
- 「進度紀錄」是**累加式**，新的加在最後一行
- 「待辦」完成就改 `[x]`，不要刪

## 更新流程

1. `ls $VAULT/Projects/` 先看現有檔案
2. 判斷要改哪一個
3. Read 讀出現有內容
4. Edit 做必要修改：
   - 進度紀錄加一行 `- YYYY-MM-DD 做了什麼`
   - 待辦加/改對應項目
   - 最新結論視情況覆寫
   - metadata 有變動才改
5. commit + push

## Commit 訊息範例

- `王品：4/8 客戶確認打包檔格式`
- `Uber Eats：第一篇 Reels 已開發票 20,000`
- `Appier：維肯設計已提供，等回覆開發票`
- `挖礦：新增第二期款項進度`

## 結案流程

當阿忠說「X 結案」「X 完成了」時：
1. 把檔案從 `Projects/` 搬到 `Archive/`（用 `git mv` 或 Write + rm）
2. 狀態改成 `已完成`
3. 加 `> 完成日期：YYYY-MM-DD`
4. 最新結論改成結案摘要
5. 所有待辦項目勾 `[x]`
6. commit 訊息：`X：<原因>，結案歸檔`
